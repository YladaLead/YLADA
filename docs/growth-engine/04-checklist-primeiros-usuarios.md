# Checklist — primeiros usuários (execução humana)

Este é o **passo a passo mínimo** para gerar aprendizado real **antes** de depender de automação ou escala. Pode ser operado pelo fundador ou time; os agentes (quando existirem) **refinam** mensagens e páginas com base no que você registrar aqui.

## Pré-requisitos

- [ ] **Um nicho ou caso** escolhido (ex.: estética — ajustar ao foco real do YLADA).
- [ ] **Link ou fluxo funcional** (cadastro, diagnóstico, contato — o que for o core da conversão).
- [ ] **Proposta em uma frase** que um leigo entenda em segundos.

## Semana 1 — contato direto

1. [ ] Listar **20–50 contatos** potencialmente alinhados ao nicho (não precisa ser frio no início: rede próxima é válida para validação).
2. [ ] Enviar **mensagem curta** com convite claro (ex.: posso te mostrar como X funciona para Y?).
3. [ ] Meta: **10–30 respostas** ou tentativas documentadas (mesmo “não” conta como dado).
4. [ ] Para quem aceitar: enviar **o mesmo link** sempre, para comparar desempenho.
5. [ ] Anotar em planilha simples:
   - texto enviado (versão A/B se possível);
   - respondeu? sim/não;
   - clicou? completou? pediu ajuda?;
   - objeções em uma linha.

## Semana 2 — fechar o loop

6. [ ] Revisar: qual mensagem gerou mais **resposta** ou **clique**?
7. [ ] Ajustar **só uma coisa** (ex.: primeira linha da mensagem **ou** headline da landing — não tudo de uma vez).
8. [ ] Repetir 20 novos contatos ou reengajar com nova versão onde fizer sentido.
9. [ ] Coletar **prova**: prints (com consentimento), depoimentos curtos, números (tempo até resposta, etc.).

## O que entregar aos agentes (quando usar o sistema)

Para o Analista / Criador / Experiência funcionarem bem, junte num único pacote:

- [ ] 3–5 exemplos reais de mensagens enviadas + respostas.
- [ ] Link da landing usada.
- [ ] Descrição em 5 linhas do fluxo após o clique.
- [ ] Objeções que mais apareceram.

## Critérios de “podemos sair da validação pura”

Ajuste os números ao seu contexto; a lógica é:

- [ ] Você repetiu a mesma promessa **várias vezes** sem reescrever o produto a cada feedback.
- [ ] Pelo menos **alguns** usuários completaram o fluxo desejado ou geraram lead de qualidade.
- [ ] Você sabe **por que** alguém disse não (preço, tempo, não entendeu, não confia, etc.).

Quando isso for verdade, o **Diretor** (humano ou agente) pode priorizar **testes de canal** ou **volume**, sempre com teto financeiro.

## Free vs pago no início

- **Free:** útil para reduzir atrito e coletar uso real **enquanto** valida mensagem e produto.
- **Pago / upgrade:** ganha prioridade quando houver **prova** e clareza de valor percebido; não como substituto de conversas iniciais.

Detalhes de limites e custo: documento `05-unit-economics-free-pago.md`.

## Responsável

| Item | Quem preenche |
|------|----------------|
| Execução das conversas | Time comercial / fundador |
| Registro de dados | Quem conduz o teste |
| Refino de copy/landing | Criador + Experiência (assistição) |
| “Seguir ou mudar de fase” | Diretor + critérios deste checklist |
